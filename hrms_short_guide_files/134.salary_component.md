# Salary Component

## Overview
Salary Component DocType for HRMS Payroll.

## Fields

### Main Fields

- `salary_component` (Data, Required): Name
- `salary_component_abbr` (Data, Required): Abbr
- `type` (Select, Required, Options: Earning
Deduction): Type
- `is_tax_applicable` (Check, Default: 1): Is Tax Applicable (Depends on: eval:doc.type == "Earning")
- `depends_on_payment_days` (Check, Default: 1): Depends on Payment Days
- `do_not_include_in_total` (Check, Default: 0): Do Not Include in Total
- `deduct_full_tax_on_selected_payroll_date` (Check, Default: 0): Deduct Full Tax on Selected Payroll Date (Depends on: eval:doc.is_tax_applicable && doc.type=='Earning')
- `disabled` (Check, Default: 0): Disabled
- `description` (Small Text): Description
- `statistical_component` (Check, Default: 0): Statistical Component
- `is_flexible_benefit` (Check, Default: 0): Is Flexible Benefit
- `max_benefit_amount` (Currency): Max Benefit Amount (Yearly) (Depends on: is_flexible_benefit)
- `pay_against_benefit_claim` (Check, Default: 0): Pay Against Benefit Claim (Depends on: is_flexible_benefit)
- `only_tax_impact` (Check, Default: 0): Only Tax Impact (Cannot Claim But Part of Taxable Income) (Depends on: eval:doc.is_flexible_benefit == 1 & doc.create_separate_payment_entry_against_benefit_claim !=1)
- `create_separate_payment_entry_against_benefit_claim` (Check, Default: 0): Create Separate Payment Entry Against Benefit Claim (Depends on: eval:doc.is_flexible_benefit == 1 & doc.only_tax_impact !=1)

*... and 10 more fields*

### Child Tables

- `accounts` (Table): Accounts
  - Options: Salary Component Account

## Workflow

### Step 1: Create Document
1. Go to HRMS Payroll → Salary Component → New
2. Fill required fields:
   - `salary_component`: Name
   - `salary_component_abbr`: Abbr
   - `type`: Type

### Step 2: Save/Submit
1. Review all fields
2. Save document

### Naming
- Auto-naming: field:salary_component

## Important Notes
- Document must be saved to be effective
