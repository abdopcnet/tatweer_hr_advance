# HR Settings

## Overview
HR Settings DocType for HRMS.

## Fields

### Main Fields

- `retirement_age` (Data): Retirement Age (In Years)
- `emp_created_by` (Select, Options: Naming Series
Employee Number
Full Name, Default: Naming Series): Employee Naming By
- `expense_approver_mandatory_in_expense_claim` (Check, Default: 1): Expense Approver Mandatory In Expense Claim
- `leave_approver_mandatory_in_leave_application` (Check, Default: 1): Leave Approver Mandatory In Leave Application
- `show_leaves_of_all_department_members_in_calendar` (Check, Default: 0): Show Leaves Of All Department Members In Calendar
- `auto_leave_encashment` (Check, Default: 0): Auto Leave Encashment
- `role_allowed_to_create_backdated_leave_application` (Link, Options: Role): Role Allowed to Create Backdated Leave Application (Depends on: eval:doc.restrict_backdated_leave_application == 1)
- `send_leave_notification` (Check, Default: 1): Send Leave Notification
- `leave_approval_notification_template` (Link, Options: Email Template): Leave Approval Notification Template (Depends on: eval: doc.send_leave_notification == 1)
- `leave_status_notification_template` (Link, Options: Email Template): Leave Status Notification Template (Depends on: eval: doc.send_leave_notification == 1)
- `standard_working_hours` (Float): Standard Working Hours
- `remind_before` (Time, Default: 00:15:00): Remind Before (Depends on: send_interview_reminder)
- `send_holiday_reminders` (Check, Default: 1): Holidays
- `send_work_anniversary_reminders` (Check, Default: 1): Work Anniversaries 
- `frequency` (Select, Options: Weekly
Monthly, Default: Weekly): Set the frequency for holiday reminders (Depends on: eval:doc.send_holiday_reminders)

*... and 17 more fields*

### Read-only Fields

- `sender_email` (Data, Read-only): Sender Email (Fetched from: sender.email_id) (Depends on: eval:doc.sender)
- `hiring_sender_email` (Data, Read-only): Sender Email (Fetched from: hiring_sender.email_id) (Depends on: eval:doc.hiring_sender)

## Workflow

### Step 1: Create Document
1. Go to HRMS → HR Settings → New

### Step 2: Save/Submit
1. Review all fields
2. Save document

## Important Notes
- Document must be saved to be effective
