# Exit Interview

## Overview
Exit Interview DocType for HRMS.

## Fields

### Main Fields

- `employee` (Link, Required, Options: Employee): Employee
- `company` (Link, Required, Options: Company): Company
- `date` (Date): Date
- `ref_doctype` (Link, Options: DocType): Reference Document Type
- `reference_document_name` (Dynamic Link): Reference Document Name
- `interviewers` (Table MultiSelect): Interviewers
- `naming_series` (Select, Options: HR-EXIT-INT-): Naming Series
- `status` (Select, Required, Options: Pending
Scheduled
Completed
Cancelled): Status
- `employee_status` (Select, Options: 
Employee Retained
Exit Confirmed): Final Decision
- `interview_summary` (Text Editor): Interview Summary

### Read-only Fields

- `employee_name` (Data, Read-only): Employee Name (Fetched from: employee.employee_name)
- `department` (Link, Read-only, Options: Department): Department (Fetched from: employee.department)
- `relieving_date` (Date, Read-only): Relieving Date (Fetched from: employee.relieving_date)
- `date_of_joining` (Date, Read-only): Date of Joining (Fetched from: employee.date_of_joining)
- `reports_to` (Link, Read-only, Options: Employee): Reports To (Fetched from: employee.reports_to)
- `designation` (Link, Read-only, Options: Designation): Designation (Fetched from: employee.designation)
- `questionnaire_email_sent` (Check, Read-only, Default: 0): Questionnaire Email Sent
- `email` (Data, Read-only): Email ID

## Workflow

### Step 1: Create Document
1. Go to HRMS → Exit Interview → New
2. Fill required fields:
   - `employee`: Employee
   - `company`: Company
   - `status`: Status

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

### Naming
- Auto-naming: naming_series:

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
