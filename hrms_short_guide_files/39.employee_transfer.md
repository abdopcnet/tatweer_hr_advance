# Employee Transfer

## Overview
Employee Transfer DocType for HRMS.

## Fields

### Main Fields

- `employee` (Link, Required, Options: Employee): Employee
- `transfer_date` (Date, Required): Transfer Date
- `company` (Link, Options: Company): Company (Fetched from: employee.company)
- `new_company` (Link, Options: Company): New Company
- `reallocate_leaves` (Check): Re-allocate Leaves
- `create_new_employee_id` (Check): Create New Employee Id

### Read-only Fields

- `employee_name` (Data, Read-only): Employee Name (Fetched from: employee.employee_name)
- `department` (Link, Read-only, Options: Department): Department (Fetched from: employee.department)
- `new_employee_id` (Link, Read-only, Options: Employee): New Employee ID

### Child Tables

- `transfer_details` (Table): Employee Transfer Detail
  - Options: Employee Property History

## Workflow

### Step 1: Create Document
1. Go to HRMS → Employee Transfer → New
2. Fill required fields:
   - `employee`: Employee
   - `transfer_date`: Transfer Date

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

### Naming
- Auto-naming: HR-EMP-TRN-.YYYY.-.#####

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
