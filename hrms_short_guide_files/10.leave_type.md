# Leave Type

## Overview
Leave Type DocType for HRMS.

## Fields

### Main Fields

- `leave_type_name` (Data, Required): Leave Type Name
- `max_leaves_allowed` (Float): Maximum Leave Allocation Allowed per Leave Period
- `applicable_after` (Int): Allow Leave Application After (Working Days)
- `max_continuous_days_allowed` (Int): Maximum Consecutive Leaves Allowed
- `is_carry_forward` (Check, Default: 0): Is Carry Forward
- `is_lwp` (Check, Default: 0): Is Leave Without Pay
- `is_optional_leave` (Check, Default: 0): Is Optional Leave
- `allow_negative` (Check, Default: 0): Allow Negative Balance
- `include_holiday` (Check, Default: 0): Include holidays within leaves as leaves
- `is_compensatory` (Check, Default: 0): Is Compensatory
- `expire_carry_forwarded_leaves_after_days` (Int): Expire Carry Forwarded Leaves (Days)
- `allow_encashment` (Check, Default: 0): Allow Encashment
- `earning_component` (Link, Options: Salary Component): Earning Component
- `is_earned_leave` (Check, Default: 0): Is Earned Leave
- `earned_leave_frequency` (Select, Options: Monthly
Quarterly
Half-Yearly
Yearly): Earned Leave Frequency
- `rounding` (Select, Options: 
0.25
0.5
1.0): Rounding
- `maximum_carry_forwarded_leaves` (Float): Maximum Carry Forwarded Leaves
- `is_ppl` (Check, Default: 0): Is Partially Paid Leave
- `fraction_of_daily_salary_per_leave` (Float): Fraction of Daily Salary per Leave
- `allow_over_allocation` (Check, Default: 0): Allow Over Allocation
- `allocate_on_day` (Select, Options: First Day
Last Day
Date of Joining, Default: Last Day): Allocate on Day
- `max_encashable_leaves` (Int): Maximum Encashable Leaves
- `non_encashable_leaves` (Int): Non-Encashable Leaves

## Workflow

### Step 1: Create Document
1. Go to HRMS → Leave Type → New
2. Fill required fields:
   - `leave_type_name`: Leave Type Name

### Step 2: Save/Submit
1. Review all fields
2. Save document

## Important Notes
- Document must be saved to be effective
