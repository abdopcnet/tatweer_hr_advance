# Employee Advance

## Overview
Employee Advance DocType for HRMS.

## Fields

### Main Fields

- `naming_series` (Select, Options: HR-EAD-.YYYY.-): Series
- `employee` (Link, Required, Options: Employee): Employee
- `employee_name` (Read Only): Employee Name (Fetched from: employee.employee_name)
- `posting_date` (Date, Required): Posting Date
- `purpose` (Small Text, Required): Purpose
- `advance_amount` (Currency, Required): Advance Amount
- `advance_account` (Link, Options: Account): Advance Account (Fetched from: company.default_employee_advance_account)
- `mode_of_payment` (Link, Options: Mode of Payment): Mode of Payment
- `repay_unclaimed_amount_from_salary` (Check, Default: 0): Repay Unclaimed Amount from Salary
- `currency` (Link, Required, Options: Currency): Currency (Depends on: eval:(doc.docstatus==1 || doc.employee))
- `exchange_rate` (Float, Required): Exchange Rate (Depends on: currency)

### Read-only Fields

- `department` (Link, Read-only, Options: Department): Department (Fetched from: employee.department)
- `paid_amount` (Currency, Read-only): Paid Amount
- `claimed_amount` (Currency, Read-only): Claimed Amount
- `status` (Select, Read-only, Options: Draft
Paid
Unpaid
Claimed
Returned
Partly Claimed and Returned
Cancelled): Status
- `company` (Link, Required, Read-only, Options: Company): Company (Fetched from: employee.company)
- `return_amount` (Currency, Read-only): Returned Amount
- `pending_amount` (Currency, Read-only): Pending Amount (Depends on: eval:cur_frm.doc.employee)

## Workflow

### Step 1: Create Document
1. Go to HRMS → Employee Advance → New
2. Fill required fields:
   - `employee`: Employee
   - `posting_date`: Posting Date
   - `purpose`: Purpose
   - `advance_amount`: Advance Amount
   - `currency`: Currency
   - `exchange_rate`: Exchange Rate

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

### Naming
- Auto-naming: naming_series:

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
