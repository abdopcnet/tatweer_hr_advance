# Leave Ledger Entry

## Overview
Leave Ledger Entry DocType for HRMS.

## Fields

### Main Fields

- `employee` (Link, Options: Employee): Employee
- `employee_name` (Data): Employee Name (Fetched from: employee.employee_name)
- `leave_type` (Link, Options: Leave Type): Leave Type
- `transaction_type` (Link, Options: DocType): Transaction Type
- `transaction_name` (Dynamic Link): Transaction Name
- `leaves` (Float): Leaves
- `from_date` (Date): From Date
- `to_date` (Date): To Date
- `is_carry_forward` (Check, Default: 0): Is Carry Forward
- `is_expired` (Check, Default: 0): Is Expired
- `is_lwp` (Check, Default: 0): Is Leave Without Pay
- `holiday_list` (Link, Options: Holiday List): Holiday List

### Read-only Fields

- `company` (Link, Required, Read-only, Options: Company): Company (Fetched from: employee.company)

## Workflow

### Step 1: Create Document
1. Go to HRMS → Leave Ledger Entry → New

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
