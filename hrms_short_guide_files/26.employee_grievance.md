# Employee Grievance

## Overview
Employee Grievance DocType for HRMS.

## Fields

### Main Fields

- `grievance_type` (Link, Required, Options: Grievance Type): Grievance Type
- `date` (Date, Required): Date 
- `status` (Select, Required, Options: Open
Investigated
Resolved
Invalid, Default: Open): Status
- `description` (Text, Required): Description
- `cause_of_grievance` (Text): Cause of Grievance
- `resolved_by` (Link, Options: User): Resolved By
- `employee_responsible` (Link, Options: Employee): Employee Responsible 
- `resolution_detail` (Small Text): Resolution Details
- `resolution_date` (Date): Resolution Date
- `grievance_against` (Dynamic Link, Required): Grievance Against
- `raised_by` (Link, Required, Options: Employee): Raised By
- `grievance_against_party` (Link, Required, Options: DocType): Grievance Against Party
- `associated_document_type` (Link, Options: DocType): Associated Document Type
- `associated_document` (Dynamic Link): Associated Document
- `subject` (Data, Required): Subject

### Read-only Fields

- `designation` (Link, Read-only, Options: Designation): Designation (Fetched from: raised_by.designation)
- `reports_to` (Link, Read-only, Options: Employee): Reports To (Fetched from: raised_by.reports_to)
- `employee_name` (Data, Read-only): Employee Name (Fetched from: raised_by.employee_name)

## Workflow

### Step 1: Create Document
1. Go to HRMS → Employee Grievance → New
2. Fill required fields:
   - `grievance_type`: Grievance Type
   - `date`: Date 
   - `status`: Status
   - `description`: Description
   - `grievance_against`: Grievance Against
   - `raised_by`: Raised By
   - `grievance_against_party`: Grievance Against Party
   - `subject`: Subject

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

### Naming
- Auto-naming: HR-GRIEV-.YYYY.-.#####

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
