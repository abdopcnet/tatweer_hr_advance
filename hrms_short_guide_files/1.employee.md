# Employee

## Overview
Employee DocType stores employee master data and basic information.

## Fields

### Basic Information
- `employee_name` (Data, Required): Full name of employee
- `company` (Link, Required): Company assignment
- `date_of_joining` (Date, Required): Employee joining date
- `status` (Select, Required): Employee status (Active/Inactive/Left)

### Organization
- `department` (Link): Department assignment
- `designation` (Link): Job designation
- `grade` (Link): Employee grade
- `reports_to` (Link): Manager/Reporting person

### Settings
- `holiday_list` (Link): Holiday list for attendance
- `user_id` (Link): Link to User account for login
- `branch` (Link): Branch/Office location

## Workflow

### Step 1: Create Employee
1. Go to HR → Employee → New
2. Enter `employee_name`
3. Select `company`
4. Set `date_of_joining`
5. Set `status` to "Active"
6. Save

### Step 2: Add Details
1. Select `department`
2. Select `designation`
3. Select `grade` (if applicable)
4. Select `holiday_list`
5. Link `user_id` (if employee needs login access)
6. Save

## Important Notes
- Employee must be "Active" to appear in payroll and leave applications
- `user_id` is required for employee self-service portal access
- `holiday_list` is used for attendance and leave calculations
