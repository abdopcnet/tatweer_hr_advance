# Shift Request

## Overview
Shift Request DocType for HRMS.

## Fields

### Main Fields

- `shift_type` (Link, Required, Options: Shift Type): Shift Type
- `employee` (Link, Required, Options: Employee): Employee
- `company` (Link, Required, Options: Company): Company
- `from_date` (Date, Required): From Date
- `to_date` (Date): To Date
- `status` (Select, Required, Options: Draft
Approved
Rejected, Default: Draft): Status
- `approver` (Link, Required, Options: User): Approver (Fetched from: employee.shift_request_approver)

### Read-only Fields

- `employee_name` (Data, Read-only): Employee Name (Fetched from: employee.employee_name)
- `department` (Link, Read-only, Options: Department): Department (Fetched from: employee.department)

## Workflow

### Step 1: Create Document
1. Go to HRMS → Shift Request → New
2. Fill required fields:
   - `shift_type`: Shift Type
   - `employee`: Employee
   - `company`: Company
   - `from_date`: From Date
   - `status`: Status
   - `approver`: Approver

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

### Naming
- Auto-naming: HR-SHR-.YY.-.MM.-.#####

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
