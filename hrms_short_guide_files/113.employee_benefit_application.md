# Employee Benefit Application

## Overview
Employee Benefit Application DocType for HRMS Payroll.

## Fields

### Main Fields

- `employee` (Link, Required, Options: Employee): Employee
- `date` (Date, Required): Date
- `payroll_period` (Link, Required, Options: Payroll Period): Payroll Period
- `company` (Link, Required, Options: Company): Company (Fetched from: employee.company)

### Read-only Fields

- `employee_name` (Data, Read-only): Employee Name (Fetched from: employee.employee_name)
- `max_benefits` (Currency, Read-only): Max Benefits (Yearly)
- `remaining_benefit` (Currency, Read-only): Remaining Benefits (Yearly)
- `department` (Link, Read-only, Options: Department): Department (Fetched from: employee.department)
- `total_amount` (Currency, Read-only): Total Amount
- `pro_rata_dispensed_amount` (Currency, Read-only): Dispensed Amount (Pro-rated)
- `currency` (Link, Required, Read-only, Options: Currency): Currency (Depends on: eval:(doc.docstatus==1 || doc.employee))

### Child Tables

- `employee_benefits` (Table): Employee Benefits
  - Options: Employee Benefit Application Detail

## Workflow

### Step 1: Create Document
1. Go to HRMS Payroll → Employee Benefit Application → New
2. Fill required fields:
   - `employee`: Employee
   - `date`: Date
   - `payroll_period`: Payroll Period
   - `company`: Company

### Step 2: Save/Submit
1. Review all fields
2. Save document
3. Submit document

### Naming
- Auto-naming: HR-BEN-APP-.YY.-.MM.-.#####

## Important Notes
- Document must be saved to be effective
- Document must be submitted to be effective
